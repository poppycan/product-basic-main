# Page Spec - 安全策略（Security Policy）

## 0. 基本信息

- **页面名称**：安全策略
- **技术栈**：Vue3 + Element Plus
- **UI 规范**：遵循 `shared/docs/页面UI规范.md`（PageHeader + PageContent、页头 actions 固定显示与选中态、主题适配）
- **页面布局**：左侧设备过滤树 + 右侧检索 / 标签统计 / 列表
- **业务定位**：基于设备（防火墙）维度过滤与检索策略数据，输出问题类型统计与列表

## 1. 页面入口（策略分析页头右侧）

- 在策略分析页的 `PageHeader` 右上角入口组新增按钮「**安全策略**」，点击跳转到安全策略管理页面。
- 路由建议：`/policy-analysis/security-policy`（可按项目路由规范调整）。
- 入口组固定显示、顺序稳定；当前路由命中时该按钮为选中态（如 `type="primary"`），其他为常态（如 `plain`）。

## 2. 页面结构

- **Header**：复用全局 PageHeader，保留「页面说明」按钮（按规范读取 `public/docs/pages/<route>.md`）。
- **Content**：左右分栏容器，左固定宽度树，右自适应。

## 3. 左侧设备过滤树

- **树层级**：父级为设备组，子级为该组下防火墙设备节点。
- **多选**：子级（防火墙）支持多选，作为右侧列表过滤条件（如 `deviceIds[]`）。
- **节点展示**：
  - 防火墙名称前显示**启/停用标识**（图标）。
  - 格式：`防火墙名称，策略总数/有问题策略数`（例：`FG-101C，150/3`）。
- **右键菜单**（仅防火墙节点触发）：
  - **设备管理**：跳转设备管理页面（URL 占位）。
  - **设备详情**：弹出设备详情弹窗（内容占位）。
  - **配置更新**：弹出配置更新弹窗（内容占位）。
  - **配置下载**：直接下载该设备配置文件。
  - **设备删除**：弹出确认弹窗，文案「确定删除选中设备？」，底部【确定】【取消】；确定执行删除，取消关闭。
- **高度与滚动**：左侧区域高度固定，内部独立滚动；不受右侧标签区域展开/收起影响。

## 4. 右侧顶部搜索栏（视觉强化）

- **检索模式**：支持**普通检索**与**高级检索**切换。
  - **普通检索**：固定字段输入——策略名称/ID、动作、源 IP、目的 IP、端口。
  - **高级检索**：单输入框，自由输入条件模糊匹配。
- **含 any 下拉**（多选，语义互斥）：紧跟输入条件后，默认无选中。选项：含 any、不含 any。
  - 含 any：过滤出含 any 项的策略。
  - 不含 any：过滤出不含 any 的策略。
- **匹配模式下拉**（单选）：在输入区最右侧，默认「**全包含**」。选项：包含输入、排除输入、相等、部分匹配。
- **重置**：提供清空图标按钮，清空所有条件。
- **触发**：检索按钮紧邻输入框，点击触发检索；输入后 Enter 触发检索。

## 5. 常用检索（搜索栏与策略标签之间）

- **设为常用**：点击常用图标，将当前条件存储为常用检索（可先 localStorage，key 如 `securityPolicy.commonSearches`，最多 5 条）。
- **展示**：最近 5 条以标签样式展示在检索框下方，浅蓝色背景。
- **删除**：支持删除单条常用检索。
- **回填与检索**：点击某条常用检索，回填到输入框并自动触发检索。

## 6. 策略标签统计区（可折叠，默认收起）

- **位置**：搜索栏下方、列表上方。
- **分类与子项**：
  - **生命周期状态**：已过期、已停用、僵尸策略、信息缺失（各子项需有定义说明，用于 tooltip）。
  - **策略质量**：冗余、冲突、隐藏、重复、违规、无效、空对象、异常策略（各子项需有定义说明，用于 tooltip）。
  - **业务标签**：用户自定义标签；默认一行自适应，超出点「更多」icon 弹出全部；行右侧「添加」icon 打开新增弹窗（标签名 + 保存）；右键业务标签弹出编辑/删除。
- **交互**：
  - 区域支持折叠/展开，**默认收起**。
  - 策略质量标题后有提示 icon，hover 显示各分类定义。
  - 标签区有设置 icon，点击弹出设置弹窗：可配置生命周期、策略质量、业务标签显示哪些子项；业务标签最多选 10 个。
- **过滤**：点击某一类下的具体项（支持多选），过滤下方列表数据。

## 7. 策略列表（复合单元格 + 行操作）

- **列与复合单元格**：每列单元格内可展示多行/多字段（如标签、名称、IP、状态等）。
- **列定义**：
  - **策略信息**：策略名称；ID｜设备（例：ID：06517｜设备：网间-亚信）。
  - **源**：源区域；源 IP。
  - **目的**：目的区域；目的 IP。
  - **服务**：服务（如 TCP_80）；动作（允许/拒绝）。
  - **状态/优先级**：状态（启用/停用）；优先级。
  - **NAT 信息**：转换前源 IP（多 IP 时 hover 显示全部）；源 IP 前小图标 hover 展示源转换规则（多规则换行，格式如「转换前IP转转换后IP:规则名称 状态」）；转换目的（IP 或 IP:端口）；目的前小图标 hover 展示目的转换规则。
  - **关联专线**：专线名称逗号分隔，超出 hover 显示全部。
  - **策略备注**：最多显示 2 行，超出 hover 显示全部；为空或缺少负责人/用途时显示警示与提示，点击补充跳转策略编辑（占位）。
  - **命中信息**：命中次数；命中分析 icon（跳转占位）；最近命中时间。
  - **标签**：以标签形式展示，每行最多 2 个，最多 2 行，超出 hover 显示全部。
  - **操作**：详情、路径分析、合规分析、策略回收、去优化、编辑、删除、查看命令（弹窗/跳转占位；合规分析点击后显示 Loading「合规分析中请稍等」）。
- **列表高度**：随标签统计区展开/收起自适应填充剩余空间，减少整页滚动。

## 8. 数据与接口（Mock 先行）

- **设备树**：`GET /mock/security-policy/device-tree` → 设备组及下属防火墙（含启用状态、策略总数、问题数）。
- **策略列表**：`POST /mock/security-policy/policies/search` → items、total、tagStats；请求体含 deviceIds、query、anyMode、matchMode、tagFilters 等。
- **常用检索**：本地 key `securityPolicy.commonSearches`，最多 5 条。
- **配置下载**：`GET /mock/security-policy/device/:id/config` 返回文件流或文本。

## 9. 主题与约束

- 使用 Element Plus 变量/主题 token，兼容浅色与深色主题。
- 左侧树固定高度、独立滚动；右侧列表高度随标签区折叠动态计算。
