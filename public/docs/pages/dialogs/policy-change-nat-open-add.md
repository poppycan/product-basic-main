# NAT 开通 — 添加 / 编辑

> 本文档为 **业务向弹窗说明**（《页面 UI 规范》§2.1.2），供弹窗「原型说明」展示：只描述功能、规则与交互，不写路由与代码。研发对照见仓库内 **技术对照** 文档。

## 用途与入口

- **新建**：在 NAT 开通列表点击 **添加** 打开；录入新的 NAT 开通意图并预览配置命令。
- **编辑**：在列表某行点击 **编辑** 打开；带入该行已有数据。是否允许修改表单内容取决于该行**当前执行状态**（见下文「只读与可编辑」）。

## 布局分区

| 区域 | 业务职责 |
| --- | --- |
| 标题栏 | 显示「NAT 开通」或「编辑 NAT 开通」；右侧 **原型说明** 打开本文。 |
| 左侧 | **开通策略表单**：按 NAT 模式采集设备、地址、转换方式、策略位置等信息。 |
| 右侧 | **配置命令**：展示根据当前表单生成的命令预览；支持更新、复制、下载（以界面按钮为准）。 |
| 底部 | **取消** 关闭不保存；**保存** 提交当前表单（需通过校验）。 |

## 只读与可编辑

- 当该行处于 **待下发** 或 **下发失败** 等允许修改的状态时：表单可编辑，可保存。
- 当处于 **已下发成功、回撤成功** 等不应再改表单的状态时：表单为只读，仅可查看命令与关闭弹窗，避免误改已生效或终态数据（具体状态名单以产品定稿为准）。

## 字段与输入规范（业务语言）

下列按界面标签理解；**切换「模式」**（如 SNAT / DNAT / NAPT / BNAT）会改变需要填写的分区与校验规则。

### 通用

- **设备**：必选；从列表选择目标防火墙（名称与地址展示）。
- **模式**：必选；决定后续显示哪些地址、接口与转换方式。
- **类型**：地址类型（如 NAT / NAT46 / NAT64），与业务场景一致。
- **策略标识**：可选「系统分配」或「自定义」；自定义时需填写符合规范的标识。
- **VR**：必选或按产品；从列表选择。
- **描述**：可选；用于备注说明。

### 随模式变化的区块

- **非 BNAT**：通常包含「原始数据」侧：源/目的安全域、源/目的地址、服务等。
- **SNAT**：强调源侧转换与出接口等（以界面分区为准）。
- **DNAT / NAPT**：强调目的转换、端口等。
- **BNAT**：包含虚拟/真实地址、BNAT 接口等专用项。

### 地址与转换（摘要）

- 源/目的地址、安全域、服务等多为 **从表格弹窗中选择** 或按规则填写；地址需符合平台校验（格式、网段等）。
- **转换方式**：如指定转换地址、出口地址等，选项随模式变化。
- **策略位置**：如置顶、置底、相对某条策略前/后；若选「相对位置」，需提供参考对象。

### 嵌套选择

- 点击「选择设备 / VR / 安全域 / 服务 / 接口」等只读输入框，会打开**选择列表弹窗**，选中后回填到当前表单；不改变主流程语义。

## 右侧配置命令

- 展示与当前表单一致的**命令预览**（联调前可能为示例拼接，以后台与设备语法为准）。
- **更新命令**：按最新表单重新生成预览。
- **复制 / 下载**：便于复制到工单或保存为文本。

## 操作与结果

| 操作 | 业务结果 |
| --- | --- |
| 取消 | 关闭弹窗，未保存的修改丢弃（以是否二次确认为准）。 |
| 保存 | 校验通过后提交；成功后关闭弹窗并应刷新列表，使新数据或变更可见。 |
| 关闭 | 与取消或右上角关闭一致，遵循产品是否提示未保存。 |
