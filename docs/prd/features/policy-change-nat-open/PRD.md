## 1. 背景与目标

### 1.1 背景

在统一纳管防火墙的前提下，安全/网络运维需要在平台侧完成 **NAT 策略的开通意图录入、命令预览、下发与执行状态跟踪**，避免在多套设备上手工核对与重复配置。

### 1.2 目标

- 提供 **NAT 开通** 独立入口（策略变更域内），支持按设备、时间、状态与多维度条件检索 NAT 开通单。
- 支持 **新增 / 编辑** NAT 开通单：按 SNAT/DNAT/NAPT/BNAT 等模式采集字段，展示 **配置命令预览**，并与列表 **执行状态、校验提示** 联动展示。
- 为后续与设备/策略服务联调预留 **列表查询、保存、预览命令** 等接口形态。

### 1.3 非目标（本期不承诺）

- 真实设备下发、配置拉取校验、与工单/审批流深度集成（除非单独需求单约定）。
- 设备树数据来源仍以纳管/CMDB 对接为准；原型阶段为 Mock。
- 与「策略开通（ACL）」共享的全部拓扑分析能力：NAT 开通以 **NAT 专用字段与命令** 为主，不复制策略开通全部分析链路，除非产品另行合并需求。

## 2. 用户、角色与场景

| 角色 | 诉求 |
| --- | --- |
| 安全策略管理员 | 创建/修改 NAT 开通单，确认命令与下发结果 |
| 网络运维 | 按设备、VR、接口检索；处理失败与回撤状态 |

**典型场景**

1. 选定防火墙 → 筛选「未完成」→ 打开一条待下发记录编辑或下发。
2. 新建 NAT：选设备与模式 → 填写地址/转换/位置 → 预览命令 → 保存。
3. 列表导出或批量删除（以实际对接为准）。

## 3. 功能范围（可验收）

- [ ] 左侧设备树单选，列表可按设备过滤（含「未选设备展示全部」语义与产品一致）。
- [ ] 时间筛选（`TimeFilterBar`）+ 状态色块（全部/未完成/已完成/已取消）+ 主查询 + 更多查询 + 查询/重置。
- [ ] 列表展示 NAT 关键字段（模式、地址、转换、执行状态、多行单元格等），分页与列表 UI 符合《页面 UI 规范》§4。
- [ ] 添加/编辑弹窗：双栏（表单 | 命令），模式切换与字段显隐正确；编辑态仅「待下发」「下发失败」可编辑（与当前实现一致）。
- [ ] 页头「页面说明」可加载页面说明文档；添加弹窗「原型说明」可加载弹窗说明文档。
- [ ] 后端就绪后：列表查询、创建/更新、删除、命令预览接口与文档草稿一致（见页面说明）。

## 4. 业务流程（主路径与异常）

### 4.1 主路径

1. 用户进入 NAT 开通页。
2. （可选）在设备树选择防火墙。
3. 设置时间、状态、查询条件 → 查询 → 浏览列表。
4. 点击「添加」→ 填写表单 → 预览命令 → 保存 → 列表出现新单或刷新。
5. 或点击「编辑」→ 修改 → 保存。

### 4.2 异常与分支

- **校验失败**：前端校验（如 IP 格式）提示 Message，不提交；后端返回业务错误时展示错误信息（联调约定）。
- **无权限**：以全局权限方案为准（PRD 不展开字段级权限，实现时补充）。
- **设备不可用 / 下发失败**：列表展示「下发失败」等执行状态；允许编辑条件以产品为准。

## 6. 数据与权限（业务级）

- **核心实体**：NAT 开通单（列表行 `NatOpenListRow`）、表单草稿（`NatOpenFormState`）。
- **设备维度**：列表与表单均关联 `deviceId` / 设备展示信息。
- **权限**（建议）：仅策略变更相关角色可访问；细粒度（只读/可下发）由权限服务与后端返回控制，前端按接口字段禁用。

## 7. 对外集成（业务级）

- **设备与 VR/安全域等**：来自纳管或资产；原型为 Mock 列表。
- **命令生成与下发**：策略/设备服务；原型为前端拼接预览，联调替换为后端预览与任务接口。

## 8. 验收标准（测试可执行）

| 编号 | Given | When | Then |
| --- | --- | --- | --- |
| AC-01 | 在 NAT 开通页 | 未选设备或选中某设备后点「查询」 | 列表数据与筛选条件一致（Mock 或接口） |

| AC-06 | 行状态为「待下发」 | 点「编辑」 | 表单可编辑，可保存 |

## 9. 风险与开放问题

- 多模式（BNAT 等）字段与真实设备命令差异大，需产品与后端共同定稿。
- 列表「导出/删除」若为占位，需在联调前替换为真实接口与二次确认文案。
