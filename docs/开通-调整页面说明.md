# 开通-调整页面说明

本文档描述「开通策略」流程中 **【调整】弹窗** 的完整逻辑，供产品、设计与开发维护；**不含代码实现**。

---

## 1. 入口与两种模式

- **触发**：列表（或等价入口）点击 **【调整】** 打开「调整」弹窗。
- **模式由「策略执行状态」决定**：

| 执行状态 | 左侧表单 | 右侧命令区 | 底部按钮 |
|---------|---------|-----------|---------|
| **下发失败**、**待下发** | **可编辑** | 完整能力（见第 7 节） | **【保存】【关闭】** |
| **下发成功**、**回撤成功**、**回撤失败** | **仅查看，不可编辑** | **仅【复制】【下载】** | **仅【关闭】**（不显示【保存】） |

- **【关闭】**：可编辑模式下为放弃本次弹窗内未保存修改并关闭；只读模式下为退出查看。

---

## 2. 布局结构

- **左侧**：开通策略表单视图（可编辑或只读）。
- **右侧**：命令视图。
- **底部**：可编辑时为 **【保存】【关闭】**；只读时为 **【关闭】** 仅一个按钮。

---

## 3. 下发方式（仅可编辑模式）

- **下发方式**（单选，必选）：**新建策略**（默认） / **可合并策略**。
- 选 **可合并策略**：弹出 **可合并策略选择**（单选、必选）→ 选后点 **【确定】** 关闭子弹窗，回显所选策略。
- **当前无可合并策略可下发**时：**「可合并策略」选项置灰**，不可切换。

---

## 4. 分支 A：新建策略（可编辑）

### 4.1 基础字段

- **策略名称**：回显外部名称；无则按产品约定规则自动生成。
- **策略 ID**：**自定义 / 系统分配**（单选必选）。选「自定义」出现文本输入；「系统分配」无输入框。
- **下发位置**（单选必选）：**置顶** / **可合并策略之前**（无可合并策略时 **置灰**）/ **插入置顶策略之前** / **插入置顶策略之后**。  
  后两项需 **指定策略**（单选）→ 弹窗选策略 → **【确定】** 后子弹窗关闭，输入框回显策略名称。

### 4.2 源信息 / 目的信息（结构相同）

- **新建对象**：第 1 个框 = 对象名（非必填，≤32 字符）；第 2 个 = 多行 **IP**。
- **引用对象**：多选，非必选。
- **区域**：必选，多选 → 弹出区域选择 → **【确定】** 回显；展示规则：逗号分隔，**最多展示 2 个名称**，超出则在第 2 个后接选中数量提示（与《页面 UI 规范》一致）。

### 4.3 服务信息（与源/目的同源，差异如下）

- **新建对象**：第 1 个 = 对象名；第 2 个 = **协议**（TCP/UDP、TCP、UDP、ICMP）单选；第 3 个 = **端口**（多行）。**端口有值则协议必填**。
- **引用对象**、**区域**：交互与展示规则同源/目的。

### 4.4 有效期、动作、备注

- **有效期**：**自定义 / 永久**（默认永久）。自定义 → 选起止时间（年月日 时分秒）→ 确定后回显；配置命令内时间对象命名可按约定生成（如 `202611112315-202711112314`）。
- **动作**：**允许 / 拒绝**（必选）。
- **备注**：默认回显外部开通策略备注，可修改。

---

## 5. 分支 B：可合并策略（可编辑）

- **左侧不展示**「新建对象 / 引用对象 / 区域」等 **可编辑块**。
- **仅展示合并后的只读摘要**（将可合并策略在源、目的、服务上唯一不同的项合并后的展示），用于理解即将下发的配置；**全部只读**。
- **策略名称**不单独编辑：**默认等于**所选可合并策略名称。
- **源安全域、目的安全域、源对象、源地址、目的对象、目的地址、服务对象、协议、端口、动作、有效期、备注** 等：**仅回显，不可编辑**。
- **业务含义**：对选中可合并策略以 **配置命令** 方式做「修改可合并策略配置」式下发。

---

## 6. 填写与匹配逻辑（新建策略下发核心）

1. **对象名称去重**：不允许重复；冲突时 **自动在名称后追加序号**。
2. **逗号**：对象内容中的逗号 **不作为** 通用「拆成多个对象」的规则；防火墙不支持跨网段时按逗号拆 IP 属于 **单独下发策略逻辑**，不是通用对象拆分规则。
3. **信息均可编辑**：对象名、IP、端口等均可改。
4. **IP 与已有对象匹配**：  
   - 匹配成功 → 回显为 **引用** 侧信息。  
   - 未匹配 → 按防火墙规则 **新建**。  
   - 若墙 **单对象支持多网段**：多个 IP 可合并为一个对象。  
   - 若 **不支持跨网段**：按逗号拆 IP 再逐条匹配；匹配 → 引用；未匹配 → 新建区展示。若 **全部未匹配**：默认可按 IP 作为名称展示，可编辑；用户改名后下发时，不支持跨网段则拆多条，名称带序号（如 B、B1…）。
5. **源 / 目的 / 服务** 展示与下发：  
   - 默认回显已 **引用** 对象，可改；点击打开 **策略对象** 弹窗：**源/目的仅 IP 地址对象**，**服务仅服务类型对象**；可不选 = 无引用下发。  
   - 未匹配部分落在 **新建** 区，可改可空；空 = 不下发新建对象。  
   - 下发组合三种：**仅引用**、**仅新建**、**引用 + 新建**。

---

## 7. 校验（可编辑模式，保存/下发前）

1. **源、目的、服务** 各自：**引用 + 新建不能同时为空**。  
   - 例（源）：「源信息为空，引用对象和新建对象至少输入一项！」
2. **新建对象有名称但无 IP/端口**（按维度提示）：  
   - 例：「源新建对象 IP 为空，不支持空对象下发！」
3. **无名称但有 IP/端口**：按防火墙是否支持跨网段，自动命名或拆对象（与第 6 节一致）。

---

## 8. 右侧命令区

### 8.1 可编辑模式

- **【更新命令】**：按左侧当前数据刷新命令区。
- **【复制】**：复制命令内容；页面顶部 Toast：**复制成功 / 失败**。
- **【下载】**：下载命令为 txt。
- **【编辑】**：进入命令文本编辑；【编辑】隐藏，显示 **【保存】【取消】**（命令区内部）。  
  - **保存**：结束编辑；**左侧表单与解析后的命令对齐**（例：命令里源新建对象 A→B，保存后左侧源新建对象名同步为 B）。  
  - **取消**：放弃本次命令编辑，命令恢复为进入编辑前的默认。
- **【翻译】图标**：占位，后续补充。

### 8.2 只读模式

- **仅【复制】【下载】**；无更新命令、无编辑、无翻译入口。

*说明：命令区内的【保存】仅作用于命令编辑子流程；弹窗底部【保存】为整单保存，二者语义不同。*

---

## 9. 弹窗底部行为（可编辑模式）

- **【保存】**：  
  - 通过校验后，**左侧已编辑内容（含已与命令同步的结果）全部生效**，弹窗关闭。  
  - **执行状态** 置为 **「待下发」**。  
  - 若保存前状态为 **「下发失败」** 且执行区已有 **下发时间**：**清空该下发时间**。
- **【关闭】**：**放弃**弹窗内未保存修改，关闭弹窗。

---

## 10. 列表外层回显（与弹窗并列）

- **仅拼接展示对象名称**；须覆盖 **新建对象名称 + 引用对象名称** 两类来源。
- **若只有一类有值**：只显示有值的那一类（仅有新建则只显示新建名称；仅有引用则只显示引用名称）。

---

## 11. 只读模式左侧

- 整表 **只读**；若需展示可合并类信息，同样为 **只读摘要**，**不出现** 新建/引用/区域等可编辑块。
