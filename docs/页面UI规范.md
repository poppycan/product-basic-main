## 1. 注意事项

- **不要过度蔓延需求**：严格按照描述要求实现，不要添加额外的功能或优化
- 保持代码风格与现有项目一致
- 保存UI风格与现有项目一致
- 要适配浅色和深色主题风格
- 如果有公共组件，则放到src\components里
- 如果功能存在配置参数，参数要清晰，易于后续调整

## 2. 新页面生成布局要求
使用GenericPage.vue生成页面
- **页面结构**：
  - 页头 (Header)
  - 页体 (Content)

- **页头功能**：
  - 右侧显示"页面说明"按钮
  - 点击按钮弹出对话框显示当前功能的说明文档（为项目全局统一的，交互完全一样，参考已有页面实现方式）

### 页面说明文档
- **文档格式**：Markdown
- **存放位置**：`public/docs/pages/` 目录
- **命名规范**：使用路由路径作为文件名（去掉前导斜杠），如：
  - `docs/pages/policy-analysis.md`
  - `docs/pages/business-performance.md`
  - `docs/pages/alert-center.md`
  - 等等

- **加载方式**：实时读取对应的 Markdown 文件内容并在弹窗中渲染显示

- **文档内容**：每个页面创建基础的 Markdown 说明文档，包含：
  - 功能概述
  - 核心需求：需求背景，该功能页面的实现目标，解决什么问题
  - 目标用户：给哪些角色的用户使用
  - 主要功能点
  - 使用说明（占位内容即可）

## 3. 页头右上角功能入口交互规范（全局）

### 3.1 适用范围
- **适用页面**：所有存在业务功能入口的页面（例如某一级菜单下包含多个常用子功能入口的场景）。
- **适用区域**：统一使用 `PageHeader` 的右上角区域（`actions` 插槽）展示功能入口。

### 3.2 显示原则（固定显示，不因点击隐藏）
- **固定显示**：同一业务域的一组功能入口在页头右上角**始终固定显示**。
- **不隐藏**：点击任意入口并跳转后，其余入口**仍保持可见与可点击**，不做隐藏/移除。
- **一致性**：同一业务域下所有子页面必须保持**同一组入口及顺序**，避免页面间入口缺失导致用户迷失。

### 3.3 状态反馈（选中态）
- **当前入口高亮**：当路由处于某个子功能页时，对应入口展示高亮样式（例如 `primary`），其它入口保持常态（例如 `plain`），但不隐藏。
- **总览页规则**：当处于业务域总览页（例如 `/policy-change`）时，默认不强制高亮某个入口（如需高亮规则可在具体业务中定义）。

### 3.4 路由与信息架构建议
- **业务域与入口组对应**：每个一级菜单对应一个业务域；常用子功能以“入口组”形式沉淀在页头右上角。
- **推荐路由结构**：
  - 业务域总览：`/xxx`
  - 子功能页面：`/xxx/a`、`/xxx/b`、`/xxx/c` ...
- **复用要求**：业务域总览页与所有子功能页应复用同一份入口配置，确保入口组统一与可维护。

### 3.5 视觉与布局要求
- **布局**：入口右对齐，水平排列，按钮间距一致；小屏下优先保持顺序，必要时缩小按钮尺寸/文字而非改变入口集合。
- **文案**：使用简洁动作型短语（例如“策略开通”“策略修改”）。
- **样式**：入口按钮不抢占主标题视觉焦点，建议使用轻量按钮风格（例如小尺寸按钮），并通过选中态区分当前入口。

### 3.6 开发实现要求（约束）
- **统一封装**：为每个业务域封装入口渲染（组件或配置驱动），供该业务域所有页面复用。
- **禁止做法**：
  - 禁止在不同子页面中维护不同的入口集合（导致入口不一致）。
  - 禁止点击入口后通过条件渲染将其它入口从 DOM 中移除或隐藏。

## 4. 列表通用 UI 规范（分页 / 高度 / 空行显示）

### 4.1 分页与行数展示（全局通用）

- **固定每页行数**：列表按分页设置显示固定行数（例如 10 条/页）。即便当前页真实数据不足该行数，也需补齐空白行，使表格在视觉上始终保持整页高度一致（例如每页固定 10 行）。
- **空行的渲染规则**：补齐的空行与真实数据行在表格结构上一致，但所有字段为空。空行不显示占位符或操作项，见 4.3。

### 4.2 自适应屏幕高度与滚动（全局通用）

- **自适应可视区域高度**：表格容器高度由布局自适应（如 flex:1），每页固定行数，保证整页视觉高度统一。大屏下当前页行数全部可见；小屏或窗口高度不足时，表格区域无法容纳当前页全部行数时，仅在表格内部出现纵向滚动条，不改变整页布局高度。
- **纵向滚动行为**：仅列表主体区域出现纵向滚动条（行内容上下滚动），分页栏、表头等保持在列表区域外，不随内容滚动。
- **横向滚动行为**：在有横向溢出时固定显示横向滚动条，高度固定为可视区域，横向滚动条始终出现在视口底部，不再依赖当前行数。

### 4.3 空行显示规范（全局通用）

- **字段列**：仅在有真实数据时渲染单元格内容（文本、标签、图标等）。对于补齐的空行，字段列保持空白，不显示「-」、0、「无数据」等占位文案。
- **操作列**：仅对真实数据行渲染操作按钮（详情、编辑、更多等）。空行的操作列不渲染任何内容，完全空白。

### 4.4 表格列规范（全局通用）

- **内容列字号统一**：除表头外，表格所有内容列的字号应统一（建议 12px），避免某列内容字号明显大于其他列。
- **操作列宽度自适应**：操作列使用 `min-width` 而非固定 `width`，使列宽根据显示的按钮数量与文案自适应；建议最小宽度约 100px（可按实际按钮数量调整）。
- **操作列左右边距一致**：操作列单元格的左右内边距应一致（如均为 12px），保证首字到左边距与末字到右边距相等；若为固定右侧列，需覆盖 Element Plus 默认样式以保持左右边距一致。
